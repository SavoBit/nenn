XML entities are XML data primitives. Parsers usually allow users to define their own:

<?xml version="1.0" standalone="yes" ?>
<!DOCTYPE author [
  <!ELEMENT author>
  <!ENTITY js "John Smith">
]>
<author>&js;</author>

--> <author>John Smith</author>

Exploitation

Some parsers allow entity definitions to point to external resources via URL—this is an external entity definition.

Weakly configured parsers naively expand external entity definitions by requesting the URL, which can lead to sensitive data disclosure, internal network access, and even occasional RCE.

Here's an example definition which could be used to disclose local files.

<?xml version="1.0" encoding="ISO-8859-1"?>
<!DOCTYPE foo [
  <!ELEMENT foo ANY >
  <!ENTITY xxe SYSTEM "file:///etc/passwd" >
]>
<foo>&xxe;</foo>

Mitigation

If you're starting something new, try to just avoid XML. Otherwise, use a secure library configured to prohibit XXE rather than rolling your own. Read the documentation to understand the security implication of what you're doing.

And as always, if it came from a user, you should distrust it by default.

For further information, consult the OWASP XXE Wiki.
